\begin{frame}{Задача 1: честное распределение ресурсов для одновременного исполнения набора Execution}

% Формула вычисления Priority,
% которая определяет приоритет исполнения каждого Execution в любой момент времени,
% а также описание всех параметров, которые в ней используются.

\small
    \begin{equation*}\label{eq:priority}
    priority(T) =
        \sqrt{
            \left(\frac{T - T_{start}}{T_{total}}\right)^2
            +
            \alpha \cdot \left(\frac{T_{total} - T_{done}}{T_{total}}\right)^2
        }
    \end{equation*}

    \begin{itemize}
    \item $T_{start}$ --- время начала исполнения Execution.
    \item $T_{total}$ --- общее ожидаемое время исполнения Execution.
    \item $T_{done}$ --- общее ожидаемое время уже исполненных Jobs.
    \item $\alpha$ --- определяет влияние реального времени исполнения и влияние ожидаемого оставшегося времени исполнения.
    \end{itemize}

\end{frame}

\begin{frame}{Задача 3: анализ подозрительности поведения участника при участии в дуэли}

\small
    В процессе участия в дуэли собираются события следующих видов:
    \begin{itemize}
    \item Ввод кода, удаление кода, перемещение курсора.
    \item Вставка кода из буфера обмена.
    \item Запуск решения на тестах из условия и произвольных тестах.
    \item Отправка решения на проверку.
    \end{itemize}

\end{frame}

\begin{frame}{Задача 3: анализ подозрительности поведения участника при участии в дуэли}

\small
    Список событий трансформируется в вектор признаков, таких как:
    \begin{itemize}
    \item Средняя скорость ввода кода, разброс.
    \item Количество вставок кода из буфера обмена, средняя длина вставляемого фрагмента.
    \item Частота изменения вставленного кода.
    \item Количество значимых перемещений курсора.
    \item Количество запусков решения и отправок на проверку.
    \end{itemize}

\end{frame}
